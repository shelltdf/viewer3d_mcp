# 用户说明书：gltf-mcp

## 环境前提

- 使用支持 WebGL 的桌面或移动浏览器。

## 获取与打开

- **开发调试**：在 `gltf-mcp/` 执行 `npm run dev`，浏览器打开终端提示的本地地址。
- **构建产物**：将 `dist/` 部署到静态站点；若需配合 Cursor MCP，请同时按开发维护说明书配置桥接与 `npm run start`（或同端口代理）。

## 使用功能

1. **从 URL 加载**：在主菜单 `File` 中输入 `.gltf/.glb` URL 后点击加载。
2. **从本机加载**：在 `File` 中选择本地文件（可多选，建议同时选 `.gltf + .bin + 贴图`）。
3. **观察与操作**：`Alt+LMB` 旋转，`Alt+MMB` 平移，`Alt+RMB` 推拉。
4. **布局切换**：左侧 `Viewport 1/2/3/4` 控制中心 panel 数量与组合。
5. **对象管理**：Outliner/Graph/3D 互相联动；点击 3D 空白会取消选择；右侧 Attribute Editor 显示当前对象属性。
6. **全屏**：按 `F10` 进入系统级全屏 3D 显示，按 `Esc` 退出。
7. **焦点框选**：按 `F` 在有场景时会平滑聚焦（约 1 秒）到当前目标；空场景时回到默认视角。
8. **渲染环境设置**：右侧 Dock 提供 `Environment / Display / Camera` 可折叠配置区，可调灯光、网格平面、辅助对象、选择线框参数等。
9. **Cursor MCP**：在 AI 中调用 `load_gltf_url` 等工具前，**本页须保持打开**。

## 常见问题

- **模型不显示 / 网络错误**：若资源在另一域名，需该服务器允许跨域（CORS）。
- **MCP 报错无法连接桥接**：确认已 `npm run dev` 且 `GLTF_MCP_BRIDGE_URL` 与端口一致。
- **离线运行（EXE）**：桌面版内置本地 Draco 解码器（`/draco/`），离线可打开本地 glTF/GLB（含 Draco 压缩）。仅当你加载远程 URL 模型时才需要网络。

## 植物工作室（`plant-studio/`）

- **打开**：在 `plant-studio/` 执行 `npm run dev` 或 `python run_web.py`，按终端地址在浏览器中打开。
- **布局**：中间为三维视口；**左侧竖条「场景」** 展开/折叠环境与光照、风；**右侧竖条「生成」** 展开/折叠生成与形态参数。折叠后仍保留缘条按钮，以免找不到入口。
- **视图菜单**：顶栏第二行 **视图** 中可勾选重新显示 **场景 Dock** / **生成 Dock**（与 `window-gui-documentation` 中「关闭后仍可恢复」一致）。
- **调宽**：侧栏展开时，靠视口一侧的 **竖条分割条** 可拖动（鼠标变为左右箭头）；窄屏下分割条隐藏，侧栏以固定最大高度滚动展示。
- **适配视图**：顶栏 **适配视图** 将相机对准当前植物。
